# Estagiliza

A modern Next.js application using React 19, Tailwind CSS, and a server-aware app router. This repository includes support for PDF processing, email delivery, theming, state management, and AI-assisted utilities.

## Project overview

This project is built with:

- **Next.js 16.2.6** — app router, server components, and optimized production build.
- **React 19.2.4** — the latest stable React version.
- **Tailwind CSS 4** — utility-first styling for fast UI development.
- **TypeScript** — static types for safer development.
- **Zustand** — lightweight state management.
- **Nodemailer / pdf-parse / pdfjs-dist** — backend email and PDF processing capabilities.
- **@google/genai** — Google generative AI integration.

## Why this README exists

This README is intended to:

## Quick start

Install dependencies:

```bash
npm install
```

Run the development server:

```bash
npm run dev
```

Open http://localhost:3000 in your browser.

## Available scripts

| Command | Purpose |
| --- | --- |
| `npm run dev` | Start the local Next.js development server |
| `npm run build` | Compile the app for production |
| `npm run start` | Start the production server after build |

## Recommended workflow

1. Create or update local environment variables in `.env.local`.
2. Run `npm install`.
3. Use `npm run dev` for local development.
4. Run `npm run build` before deploying.
5. Run `npm run lint` to catch issues early.

## Environment variables

Keep secrets and local configuration out of Git by using `.env.local`.

Examples of values typically stored here:

- API keys
- service credentials
- SMTP configuration
- third-party integration secrets

> Do not commit `.env.local` or other local environment files to GitHub.

## Recommended project structure

- `app/` — Next.js app routes and server/page components.
- `components/` — reusable UI pieces.
- `lib/` — utility functions and shared logic.
- `public/` — static assets and images.
- `styles/` or Tailwind config files — styling configuration.

## Git ignore rationale

The `.gitignore` file is designed to keep the repository clean and safe by excluding:

- build artifacts (`.next/`, `out/`, `build/`)
- installed dependencies (`node_modules/`)
- local environment secrets (`.env.local`, `.env.*.local`)
- editor/project metadata (`.vscode/`, `.idea/`)
- OS temporary files and logs
- machine-generated TypeScript cache files

This prevents accidental commits of files that are:

- platform-specific,
- large and regeneratable,
- sensitive or private,
- not required to build the app from source.

## Deployment

For deployment, use a production build:

```bash
npm run build
npm run start
```

If you deploy to Vercel, the service will automatically detect this Next.js app.

## Notes

- `package-lock.json` should stay committed to preserve reproducible installs.
- `next-env.d.ts` is generated by Next.js and should not be tracked in source control, but it can be regenerated automatically.
